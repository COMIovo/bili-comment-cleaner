# B站评论清理助手

这是一个 Chrome / Edge Manifest V3 扩展，用 AICU 的历史评论索引定位当前登录 B站账号的过往视频评论，再通过 B站登录态进行预览、核验、勾选删除和日志导出。

## 当前能力

- 读取当前浏览器的 B站 `DedeUserID` 与 `bili_jct`，不保存账号密码。
- 从 `api.aicu.cc` 分页拉取当前 UID 的历史评论索引。
- 面板将“AICU 全历史扫描”和“指定视频直扫”拆成两个默认展开的二级菜单，可按需收起。
- 指定视频直扫只需要粘贴 BV、av 或视频链接，会直接扫描该视频当前评论区，并默认只保留当前登录 UID 发布的一级评论和二级回复。
- 普通视频输入会先标准化，只保留 `BV...` 或 `av...`；链接里的 `#reply`、查询参数和追踪参数会被丢弃。
- 全历史扫描会忽略指定视频输入；只有点击“直扫指定视频”时才使用 BV / av / 视频 URL。
- 在手动导入框里只粘贴 BV / 视频链接时，会自动直扫视频评论区，结果仍放在手动导入预览区，不再让小白卡在 rpid/root 概念上。
- 手动导入框里的视频链接即使带 `#reply` 评论锚点，也会按“扫描整条视频”处理，避免把别人 @ 你的回复当作候选。
- 指定视频直扫必须读到评论作者 UID 且等于当前登录 UID 才会加入候选，别人 @ 你的评论不会被当作你发的评论。
- 手动导入拥有独立功能页和独立预览删除区，导入项不会混入 AICU 预览区。
- 手动导入只有 `rpid` 但缺少父评论 `root` 时会先标记为“待核验”，删除前会尝试用 `seek_rpid` 和二级回复分页反查真实 root。
- 手动导入支持 `aid,rpid,root` 三数字格式，可直接指定二级回复的父评论。
- 手动导入不会再把 BV 号中的数字误识别成评论 `rpid`；只有明确提供评论 ID 才会导入。
- 删除前会优先尝试 `reply/info` 精确核验评论，再回退到详情页、`seek_rpid` 和二级回复分页。
- 导入 BV 链接时会尝试向 B站查询 aid，保证删除接口参数完整。
- 支持关键词、日期范围、指定视频筛选。
- 支持一级评论和二级回复候选项。
- 删除前默认调用 B站评论详情接口核验评论仍属于当前 UID。
- 删除时按间隔逐条执行，遇到未登录、CSRF 错误或请求频繁会停止。
- 支持 CSV / JSON 导出完整候选和处理日志。
- 成功删除、已不存在、归属不符的评论会写入本地已处理记录库，后续扫描默认隐藏，避免重复跳过。
- 可切换“显示已处理评论”，也可清除当前 UID 的本地已处理记录。
- B站接口请求会通过一个 `bilibili.com` 工作标签页执行，尽量避免扩展页跨站请求不携带登录 cookie 的问题。
- AICU 扫描请求会通过一个 `aicu.cc` 工作标签页执行，避免扩展页来源触发 `HTTP 468`。
- AICU 扫描恢复直连为主路径，工作页/API 页签只作为失败兜底。
- 如果 AICU 工作页脚本没有返回结果，会自动改用 `api.aicu.cc` 页签读取 JSON。
- 如果 AICU 工作页本身加载成错误页，也会继续尝试 `api.aicu.cc` 页签兜底。
- 临时创建的 `www.aicu.cc` 工作页会在读取完成后自动关闭；如果你已经手动打开了 AICU 页面，插件会复用它但不会关闭它。
- 临时创建的 `api.aicu.cc` 兜底页签会在读取完成后自动关闭。
- 可复制诊断日志，方便排查删除接口返回码、核验失败和过期索引。
- 对 AICU 返回的对象型 `root` 做兼容处理，并在严格核验时使用 `seek_rpid` 兜底。

## 安装方法

1. 打开 Edge 或 Chrome 的扩展程序页面。
2. 开启开发者模式。
3. 点击“加载已解压的扩展程序”。
4. 选择本目录 `bili-comment-cleaner`。
5. 登录 B站后点击扩展图标，进入独立管理页。

## 使用建议

1. 先刷新登录态，确认显示当前 UID。
2. 设置关键词、时间范围，选择 AICU 全历史扫描或指定视频直扫。
3. 指定视频直扫只需要粘贴 BV / av / 视频链接，插件会自动使用当前登录 UID 查找你的评论。
4. 检查“扫描预览与删除”里的候选。
5. 如需脱离扫描流程，可在“手动导入功能页”里粘贴评论链接、CSV 或 JSON，并在其二级预览区单独处理。
6. 只在对应功能页里勾选确定要删除的评论。
7. 保持默认严格核验和较慢删除间隔。
8. 删除结束后导出 CSV 或 JSON 留档。

## 边界说明

- 第一版只支持普通视频评论，评论区类型固定为 `type=1`。
- AICU 数据不是实时数据，可能存在漏评论或已删除评论仍在索引里的情况。
- 手动导入只能提供候选定位，真正删除前仍以 B站核验结果为准。
- 已处理记录仅保存在本机浏览器的扩展存储中，更换浏览器或清空扩展数据后需要重新积累。
- 严格核验失败的评论会被跳过，不会继续删除。
- B站接口和 AICU 接口都可能变动，如出现大面积失败，需要更新适配逻辑。
